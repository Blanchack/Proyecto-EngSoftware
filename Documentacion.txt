Caso de uso UC1: Ingresar Solicitud
Actor principal: Vecino
Actores involucrados:
  - Vecino: ingresar la solicitud
  - Servicio de geolocalización: verificar la ubicación del vecino
Precondiciones:
  - El vecino debe ser un usuario verificado.
Post condiciones:
  - La cooperativa visualiza la solicitud correctamente.
  - Vecino obtiene un recibo con la información del caso ingresado exitosamente
Escenario principal de exito (flujo básico)
  1. El vecino ingresa su solicitud de reciclaje.
  2. El servicio de geolocalización verificar la ubicación del vecino.
  3. Con la respuesta del servicio, el sistema asigna el caso a la cooperativa que corresponda.
  4. La cooperativa recibe con exito la solicitud para su procesamiento.
Extensiones:
  3' El sistema recibe una señal de error desde el servicio de geolocalización
    - El sistema intente denuevo hasta 3 veces.
    - Si el error persiste se le notifica al vecino para intentar denuevo
    
  

Caso de uso UC2: Revisar la ruta
Actor principal: Conductor
Actores involucrados: 
  - Cooperativa: Confirmar ruta para el conductor
  - Conductor: Revisar la ruta
Precondiciones:
  - La cooperativa visualiza varias opciones de rutas
Postcondiciones:
  - El conductor puede ver la ruta a seguir.
Escenario principal de exito (flujo)
  - La cooperativa escoge la ruta a seguir
  - Se le envia la ruta al conductor
  - Conductor recibe la ruta correctamente.
Extensiones:
  1' Problemas en la ruta
  - El conductor da aviso de algun problema en la ruta a la cooperativa
  - La cooperativa revisa alternativas para la ruta
  - Se actualiza la ruta del conductor
