Caso de uso: Ver solicitudes
Actor principal: Vecino
Actores involucrados: 
	-Vecino quiere revisar las solicitudes ingresadas
Precondiciones:
	-Vecino debe tener un perfil verificado para poder ingresar solicitudes
	-Vecino debe tener alguna solicitud para poder revisarla
Postcondiciones:
	-Vecino es capaz de ver todas las solicitudes, opcionalmente puede:
		-Revisar el estado de una solicitud
		-Ver el recibo de una solicitud
		-Filtrar la lista de solicitudes, en caso de tener varias
Flujo principal UC "Ver solicitudes":
	1.Vecino solicita revisar las solicitudes
	2.Sistema carga las solicitudes asociadas al vecino
	3.Vecino es capaz de visualizar las solicitudes
	
	Extensiones:
	3'.Vecino quiere ver recibo: ejecutar caso de uso "Ver recibo"
	3'.Vecino quiere filtrar las solicitudes: ejecutar caso de uso "Filtrar"
	3'.Vecino quiere ver el estado de una solicitud: ejecutar el caso de uso "Ver estado"
